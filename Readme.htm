ISpySoft Case Study

This document describes the content of the package that accompanies our book "Practical Software Factories in .Net". It contains parts of the Software Factories case study that we conducted in course of writing. For a quick overview, we structured this document by chapter, as it appears in the book.

Current State

This drop contains the three tools that we developed throughout the ISpySoft case study:
  • Feature Modeling DSL
  • Software Factory Schema DSL
  • Feature Model Configurator

Because we decided to publish the ISpySoft Software Factory as a community project on Codeplex and it therefore becomes a moving target, we do not to include it in this package. We rather refer you to the ISpySoft site at Codeplex to download the latest bits from there. On the Codeplex site you can find the prototypical ISpySoft application, guidance packages, and the Software Factory template. Furthermore, in the future we will also update the DSLs to use the latest version of Microsoft DSL Tools.

So stay tuned and also check out new information at www.ispysoft.net

Prerequisites

Each DSL comes with its own .MSI installer, which also contains the redistributable libraries for the DSL Toolkit. Therefore, if you only want to look at the models you will only need to install the corresponding DSLs.

In case you would like to take a look a the meta-model and possibly make your own modifications, you will need the following prerequisites installed in order to be able to open and build the source code:

Visual Studio 2005 SDK - V1 RTM October 2005
(available from the Visual Studio Industry Partner Affiliate homepage)

Domain-Specific Language Toolkit
(available from the Domain-Specific Language Tools homepage)

Chapter 3

Feature Model DSL

The Feature Model DSL allows for creating feature models as they are used to describe the commonalities and variabilities of a product line. The link below starts the installer that installs the Feature Model DSL into the Visual Studio 2005 environment.

Feature Model DSL Installer (.msi)

ISySoft Problem and Solution Feature Models

The ISpySoft is contains two feature models; one for the problem domain of Private Eye Investigation, and one for the solution domain. The two links below open the feature models in Visual Studio 2005. Requirement is that the Feature Modeling DSL is correctly installed.

ISpySoft Problem Feature Model (.featuremodel)
ISpySoft Solution Feature Model (.featuremodel)

Chapter 5

Software Factory Schema DSL

The Software Factory Schema DSL allows for creating Software Factory Schemas that are used to formally describe Software Factories. The link below starts the installer that installs the Software Factory Schema DSL into the Visual Studio 2005 environment.

Software Factory Schema DSL Installer (.msi)

ISySoft Software Factory Schema Project

The link below opens a Visual Studio 2005 solution that contains the ISpySoft Software Factory schema. Furthermore this solution contains an instance of the Software Factory Schema Browser. Upon pressing F5, Visual Studio will open a Internet Explorer window with the ISpySoft Software Factory Schema Browser

ISpySoft Software Factory Schema Project (.sln)

Chapter 8

Feature Model Configurator

The Feature Model Configurator is used during application development to select and deselect variabilities in feature models. These configurations can then be used e.g. by automated guidance steps to generate the appropriate code or configure components and services correctly.

Software Factory Schema DSL Installer (.msi)
