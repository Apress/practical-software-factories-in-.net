Software Factory Schema DSL

Installation

For installation of the Software Factory Schema DSL simply double-click on the SoftwareFactorySchemaDSL.msi, which you can find in the current directory and follow the on-screen instructions.

Getting started

  • Install the Software Factory Schema DSL.
  • Open Visual Studio 2005.
  • Select New->Project.
  • Under Visual C# category select the SoftwareFactorySchemaProject and enter the Name for the language, the Location on the hard drive and the Solution Name.

  • In the solution explorer double click on the .sfschema file.

  • After opening the empty Software Factory Schema document the Toolbox shows the concepts provided by the Software Factories Schema DSL

Elements of a Software Factory Schema

In order to create your schema simply drag the elements from the toolbox to the document. With a right click on the document surface you can check the correctness of the diagram. Errors will be shown in the output window. The different elements are explained below.

Viewpoints

You can add a new viewpoint from the Toolbox onto the schema document.

With a right click on the viewpoint you can add a new Stakeholder and in the Properties Window you can change the name to something meaningful.

Artifacts

The different kinds of Artifacts (Tools, Work Products and Assets) added to the schema by simply dragging and dropping an artifact onto the schema surface. With a double click on the artifact you can change the name to something more meaningful than the default name and you can select the type of artifact from the selection menu.

After selecting the artifact type the decorator in the shape will show you what artifact type it is (A for asset, W for Work Product and T for tool).

Activities and Mappings

The same way you added artifacts and viewpoints you can add activities and mappings to the diagram.

In the case of mappings you can double click on the shape and a dialog opens that allows you to add a mapping description to the shape. This description is currently only text but we can also think of scenarios where this description is more formal and can be used to generate other output like transformation scripts from it.

Connectors

Connections are implemented so only correct connections can be made. The table below shows an overview of allowed connections. It is important to mention that this is a directed graph. For the table this means that the left hand column is the source of the connection and the right hand columns the targets. Some connections are only allowed to be made in one pre-defined direction. In some cases the tool will ask for additional information from the user to make the connections valid (for example it will ask if the connection is read or write connection).

The Software Factory Schema Browser

 

In order to open the Software Factory Schema Browser you will need to execute the following steps:

  1. Save the Software Factory schema model.
  2. In Solution Explorer right-click SoftwareFactorySchemaBrowser.t4 and choose Run Custom Tool. This will regenerate SoftwareFactorySchemaBrowser.xml, which is a child of its T4 template. Alternatively you can use the Transform All Templates feature, which is accessible through a toolbar button in Solution Explorer.
  3. Press F5 to start the project. Visual Studio will open Internet Explorer with the Software Factory Schema Browser loaded. Note that the XML file references a stylesheet, which provides the correct HTML formatting.
